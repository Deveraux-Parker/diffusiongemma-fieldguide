Measured speed

Prefill and generation, by the numbers.

On a single RTX 4090 (fp8 KV, single-user). Prefill = time to read your prompt before generation starts; generation = pure decode, measured from the first output token.

context prefill (TTFT) prefill rate generation (decode)
2,000 tok0.44 s~4,500 tok/s~570 tok/s
8,000 tok0.93 s~8,500 tok/s~540 tok/s
16,000 tok1.94 s~8,200 tok/s~525 tok/s
Generation is content-dependent. The diffusion sampler runs more denoising steps for less-predictable text, so decode is a range — ~525–575 tok/s for steady prose, down to ~305 tok/s for dense, high-entropy output. Throughput rises with output length (it works in 256-token canvases). fp8 KV costs no speed vs fp16 — it just buys you more context.
Getting the best from it

Ask in sentences.

One small habit gets the most out of the diffusion sampler: ask for answers as a short sentence or JSON rather than a bare token, then parse the value out. It keeps responses crisp and reliable.

Prompt
“State the code from the RECORD line in one short sentence.”
Response
“The code on the RECORD line is CYAN-4821.”
Run it

To 18k context on one 4090.

Two ready configs. Both set --max-num-batched-tokens ≥ --max-model-len (single-pass prefill, clean retrieval). fp8 KV gives you more context and faster decode.

Comfortable — 10k / fp16

vllm serve … --served-model-name dg-awq \
  --max-model-len 10000 --max-num-seqs 1 \
  --gpu-memory-utilization 0.88 \
  --max-num-batched-tokens 10240 \
  --kv-cache-dtype float16

Max context — 18k / fp8

vllm serve … --served-model-name dg-awq \
  --max-model-len 18000 --max-num-seqs 1 \
  --gpu-memory-utilization 0.90 \
  --max-num-batched-tokens 18432 \
  --kv-cache-dtype fp8

Call it

curl localhost:8001/v1/chat/completions -d '{
  "model": "dg-awq",
  "messages": [{"role":"user","content":
    "...thousands of tokens of context...
     One line is a RECORD with a code.
     State that code in one short sentence."}],
  "temperature": 0, "max_tokens": 60 }'
fp8 isn't a tax. Half-size KV cache costs no measurable decode speed vs fp16 (both ~525–575 tok/s) and no retrieval-quality loss — it just buys you more context (8k → 18k on a 24 GB card). Use it whenever you want headroom.
